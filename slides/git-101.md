# Git GUI ソフトウェアを使用するメリット

- gitコマンドによるCLI操作と比べて初心者に優しい
- IDE付属のバージョン管理機能よりも高機能でシームレスな操作が可能
  - 関心領域の違いからIDEのコード記述用ウィンドウレイアウトだとイマイチ
  - IDEでコードを開いたままGitの操作ができる（マルチディスプレイで◎）

# 最低限覚えたいGitの基本操作

1. リモートリポジトリをローカルリポジトリとして複製 (clone)
2. ローカルリポジトリにコードの変更を反映 (commit)
3. リモートリポジトリにローカルリポジトリの変更を反映 (push)
4. リモートリポジトリの変更を取得 (fetch)
5. リモートリポジトリの変更を取得して取り込み (pull)
6. ブランチの作成 (branch)
7. 作業ブランチの切替 (checkout/switch)

---

# Gitを活用するために知っておきたい応用操作

8. リモートリポジトリを別のリモートリポジトリに複製 (fork)
9. 対象ブランチの変更を作業ブランチに取り込み (merge)
10. 派生する変更やブランチ(枝)を接ぎ木(リベース)して1本にする (rebase)
11. 対象のコミットを作業ブランチに取り込む (cherry-pick)
12. 作業ツリーの変更(未コミット/未ステージ)を特別な領域に退避する (stash)
13. コンフリクト(変更の競合)の解消
